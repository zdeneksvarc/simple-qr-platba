# Simple QR Platba

Simple QR Platba is a small HTTP API that generates framed `image/png` QR payment codes from URL query parameters.

It is designed for invoicing systems such as Invoice Ninja: call one image URL and embed the returned PNG directly in an invoice, email, or web page.

## Quick Start

Start the service with Docker Compose:

```sh
docker compose up --build
```

The API will be available at:

```txt
http://localhost:3000
```

Check that the service is running:

```sh
curl 'http://localhost:3000/'
curl 'http://localhost:3000/healthz'
```

Both endpoints return:

```json
{"status":"ok"}
```

Stop the service:

```sh
docker compose down
```

## QR Endpoint

By default, QR PNGs are generated by:

```txt
GET /image
```

You can change this path with the `ENDPOINT` environment variable:

```yaml
environment:
  ENDPOINT: "/abcd"
```

Rules:

- `ENDPOINT` is optional; default is `/image`.
- `ENDPOINT` must start with `/`.
- `ENDPOINT` cannot be `/` or `/healthz`.
- Unknown endpoints return `404`.

Always wrap URLs in quotes when using `curl`. Query parameters contain `&`, which shells interpret specially when the URL is not quoted.

## Generate a QR Payment

### Custom Account Details

Use this when you want to pass the bank account and currency directly in the request:

```sh
curl -o qr-custom.png \
  'http://localhost:3000/image?accountNumber=1234567890&bankCode=0800&amount=123.45&currency=CZK&vs=2026001&ks=0308&rn=ACME%20s.r.o.&message=Invoice%202026001&size=300'
```

This creates `qr-custom.png` in the current directory.

### Preset Account Details

Presets keep account number, bank code, and currency in configuration, so invoices only need dynamic values such as amount and invoice number.

The included `compose.yml` contains two demo presets:

```yaml
PRESET: "DEMO|1234567890|1234|CZK,DEMO2|1234567890|4321|EUR"
```

Preset format:

```txt
name|accountNumber|bankCode|currency
```

Multiple presets are separated by commas. Preset names are case-insensitive, so `demo`, `DEMO`, and `Demo` are equivalent.

Use the `DEMO` preset:

```sh
curl -o qr-demo.png \
  'http://localhost:3000/image?preset=demo&amount=123.45&vs=2026001&ks=0308&rn=ACME%20s.r.o.&message=Invoice%202026001&size=300'
```

Use the `DEMO2` preset:

```sh
curl -o qr-demo2.png \
  'http://localhost:3000/image?preset=DEMO2&amount=123.45&vs=2026002&message=Invoice%202026002&size=300'
```

## Query Parameters

| Parameter | Required | Description |
| --- | --- | --- |
| `preset` | No | Preset name from `PRESET`. When used, account number, bank code, and currency come from the preset. |
| `accountNumber` | Yes, without `preset` | Czech account number, optionally with prefix, for example `12-1234567890`. |
| `accountPrefix` | No | Czech account prefix when not included in `accountNumber`. |
| `bankCode` | Yes, without `preset` | Four-digit Czech bank code. |
| `amount` | No | Payment amount, zero allowed, max 10 characters after normalization, max 2 decimal places. Decimal comma is accepted and spaces are stripped, for example `0`, `123.45`, `144,44`, or `10 234,44`. |
| `currency` | No, ignored with `preset` | Three-letter currency code, for example `CZK` or `EUR`. |
| `vs` | No | Variable symbol, max 10 digits. Sent as `X-VS` in SPAYD. |
| `ss` | No | Specific symbol, max 10 digits. Sent as `X-SS` in SPAYD. |
| `ks` | No | Constant symbol, max 10 digits. Sent as `X-KS` in SPAYD. |
| `rn` | No | Recipient name, max 35 characters. Sent as `RN` in SPAYD. |
| `rf` | No | Recipient reference, max 16 digits. Sent as `RF` in SPAYD. |
| `pt` | No | Payment type, max 3 characters, for example `IP` for instant payment. Sent as `PT` in SPAYD. |
| `message` | No | Payment message, max 60 characters. Sent as `MSG` in SPAYD. URL-encode spaces as `%20`. |
| `size` | No | Requested PNG size in pixels. Default is `512`; allowed range is `120` to `2048`. The service may return a slightly larger PNG if the QR payload cannot fit into the requested size. |

## Invoice Ninja Example

Use a publicly reachable URL in the Invoice Ninja template:

```html
<img
  src="https://your.domain.com/image?preset=demo&amount=$amount_raw&vs=$number&ks=0308&message=$number $client.name&size=250"
/>
```
